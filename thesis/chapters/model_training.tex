\chapter{模型训练与优化方法}

\section{基础模型与任务形式定义}

本文以 Qwen2.5-7B-Instruct 作为基础模型，围绕股票短期涨跌方向预测任务进行监督微调与后续优化设计。选择指令模型作为基础模型的原因在于：该类模型已经具备较好的中文理解、指令遵循和结构化输出能力，可以直接接受自然语言形式的金融任务描述、K 线特征说明以及新闻和研报摘要，并按照指定格式生成预测结果。

从任务形式上看，本文不将股票预测建模为精确涨跌幅回归问题，而是将其定义为涨跌方向分类问题。对于单一预测周期，模型输出空间为“涨”和“跌”两类；对于多周期预测，模型需要同时输出未来 1 个、3 个和 7 个交易日后的方向结果。这样的任务定义更符合本文实验目标：在噪声较大的股票日频数据中，相比要求模型精确预测收益率数值，判断方向更稳定，也更便于构建监督标签和评估指标。

在具体实现中，本文形成了两类训练与验证任务。第一类是仅基于历史行情的 7 日 K 线方向预测任务，输入为连续 7 个交易日的归一化 K 线特征，输出为第 8 个交易日相对前一交易日的涨跌方向。第二类是融合外部信息的多周期预测任务，输入为目标日期前 5 个交易日的归一化 K 线、近期新闻摘要和研报摘要，输出为未来 1、3、7 个交易日相对目标日期收盘价的涨跌方向组合。前者主要用于构建稳定的监督微调主线，后者用于验证多源信息接入对预测任务的增益。

\section{监督微调数据构建}

\subsection{历史样本组织方式}

监督微调数据主要由行情数据和文本增强数据两部分构成。行情数据来自 \path|exports/quotes.csv|，其中每条记录包含股票代码、交易日期、开盘价、最高价、最低价、收盘价、成交量、成交额、振幅、涨跌幅、涨跌额和换手率等字段。为了构建可训练样本，系统按照股票代码分组，并在每只股票内部按交易日期升序排列，形成一条连续的交易日序列。

在 7 日 K 线数据集中，\path|scripts/dataset/build_quotes_7d_dataset.py| 采用滑动窗口方式生成样本。对于某只股票的时间序列，系统取连续 7 个交易日作为输入窗口，记为 Day1 至 Day7；紧接着的第 8 个交易日作为标签日，记为 Day8。若 Day8 的 \texttt{pct\_change} 大于或等于 0，则标签设为“涨”，否则设为“跌”。样本只在标签日满足指定时间范围时输出，默认训练窗口为 2021 年 1 月 1 日至 2026 年 1 月 1 日之前。

为了减少不同股票之间绝对价格、成交量规模差异带来的干扰，脚本对每只股票独立计算归一化参数。开盘价、最高价、最低价、收盘价和换手率采用 Min-Max 归一化；成交量先经过 \(\ln(1+x)\) 变换后再进行 Min-Max 归一化；涨跌幅采用 Z-score 标准化；振幅则保留原始数值。归一化参数仅基于该股票在训练标签窗口内的数据估计，并在该股票所有训练样本中复用。

最终生成的基础行情数据集为两列 CSV：\texttt{prompt} 和 \texttt{label}。其中 \texttt{prompt} 包含 7 行归一化 K 线描述和明确的任务说明，要求模型只输出“涨”或“跌”；\texttt{label} 则保存真实方向标签。当前仓库中的 \path|train/dataset/quotes_7d_pre2026_dataset.csv| 共包含 339287 条训练样本，构成本文行情方向预测的基础监督数据。

\subsection{有无外部信息的样本构造}

为了比较外部信息对预测效果的影响，本文构建了“无外部信息”和“有外部信息”两类样本。无外部信息样本仅包含历史 K 线特征，模型需要完全依赖价格、成交量和涨跌幅等结构化行情信息进行判断。这一设置对应 \path|build_quotes_7d_dataset.py| 生成的 7 日 K 线分类数据，也可以作为判断多源文本是否带来增益的基线。

有外部信息样本则在行情特征之外加入新闻和研报摘要。相关构造由 \path|scripts/dataset/build_quotes_summary_5d_dataset.py| 完成。该脚本读取 \path|exports/quotes.csv| 中的行情数据，同时从 \path|Summary/news/| 和 \path|Summary/report/| 中读取已经由大语言模型生成的新闻和研报摘要。对于每个目标日期，系统保留目标日期及之前连续 5 个交易日的归一化 K 线，同时检索截至该日期最近 30 日内的新闻摘要和研报摘要。新闻最多保留 30 条，研报最多保留 3 条。

多源样本的标签为未来 1、3、7 个交易日后的方向组合。具体而言，系统将目标日期收盘价与未来第 1、第 3 和第 7 个交易日收盘价分别比较，若未来收盘价大于或等于当前收盘价，则对应标签为“涨”，否则为“跌”。三个标签按照顺序组合为形如“涨，跌，涨”的字符串。这样可以在同一条样本中同时考察模型对不同时间尺度走势的判断能力。

此外，本文还采用批量调用大模型的方式构建带推理过程的监督数据。脚本 \path|scripts/batch/batch_qwen_cot_quotes_dataset.py| 会读取基础 7 日 K 线数据集，将原始 \texttt{prompt} 与真实标签一起组织为批处理请求，调用 Qwen 模型生成一段能够自然推出标准答案的中文推理过程，并在最后一行输出真实标签。随后脚本将结果合并为 \texttt{prompt} 与 \texttt{completion} 两列，其中 \texttt{prompt} 改写为要求模型先分析再输出最终答案的 CoT 风格提示词，\texttt{completion} 保存模型生成的推理过程与最终答案。当前 \path|train/dataset/quotes_7d_cot_from_batch.csv| 共包含 599871 条样本，用于后续 LoRA 监督微调。

\section{监督微调训练设计}

\subsection{微调目标}

本文采用监督微调使基础模型学习两类能力：第一，理解归一化 K 线特征与涨跌方向标签之间的映射关系；第二，按照固定输出约束生成可解析的预测结果。对于 CoT 风格数据，模型不仅要输出最终方向，还要在指定标记之间给出一段简短分析，从而提升输出的可读性和任务稳定性。

训练目标仍然是自回归语言模型的条件生成目标。给定用户侧 \texttt{prompt}，模型需要生成助手侧 \texttt{completion}。在训练时，损失函数只作用于助手回复部分，用户输入部分的标签被设置为 \(-100\) 进行屏蔽。这样可以避免模型把损失浪费在复现输入提示词上，而是集中学习如何在给定任务描述和特征之后生成符合格式的预测回答。

\subsection{训练数据格式}

SFT 脚本 \path|train/scripts/sft/train_sft_qwen_lora.py| 要求训练文件为 CSV 格式，并包含 \texttt{prompt} 和 \texttt{completion} 两列。对于每条样本，脚本将 \texttt{prompt} 作为用户消息，将 \texttt{completion} 作为助手消息，再通过 Qwen tokenizer 的 chat template 转换为完整对话文本。该做法与指令模型的原始对话格式保持一致，能够减少训练格式与推理格式之间的偏差。

在分词过程中，脚本分别构造“仅用户输入”的 prompt token 序列和“用户输入 + 助手回复”的完整 token 序列。完整序列减去 prompt 部分后得到 response token，并将 prompt 部分的 label 全部设为 \(-100\)。若完整序列超过最大长度，则优先保留回复部分，并从 prompt 左侧截断较早的输入 token。当前脚本默认最大长度为 2048 tokens。

为了提高多卡训练效率，脚本还会根据训练文件、模型路径、最大长度和验证集设置构建 tokenized cache。Rank 0 负责将原始 CSV 映射为分词后的 Hugging Face Dataset 并写入缓存，其余进程等待缓存完成后再加载。该机制避免了多进程重复分词，也减少了长时间训练前的数据准备开销。

\subsection{训练参数与实现设置}

本文采用 LoRA 方式进行参数高效微调，而不是对 Qwen2.5-7B-Instruct 进行全参数训练。训练脚本通过 PEFT 为基础模型注入 LoRA adapter，默认秩 \(r=64\)，\(\alpha=128\)，dropout 为 0.05，目标模块包括 \texttt{q\_proj}、\texttt{k\_proj}、\texttt{v\_proj}、\texttt{o\_proj}、\texttt{gate\_proj}、\texttt{up\_proj} 和 \texttt{down\_proj}。这种设置覆盖注意力投影和前馈网络关键线性层，在显著减少可训练参数的同时保留较强的任务适配能力。

训练超参数由 \path|train/scripts/sft/train_sft_qwen_lora.py| 和启动脚本 \path|train/scripts/launch/run_sft_qwen_lora_8gpu.sh| 共同控制。默认基础模型为 \texttt{Qwen/Qwen2.5-7B-Instruct}，训练轮数为 2，学习率为 \(2\times10^{-4}\)，学习率调度为 cosine，warmup ratio 为 0.03，单卡 batch size 为 1，梯度累积步数为 16。启动脚本默认使用 8 张 GPU 通过 \texttt{torchrun} 启动训练，因此有效 batch size 为 \(1 \times 16 \times 8 = 128\)。训练过程中启用 bfloat16、gradient checkpointing 和 safetensors 保存，以降低显存占用并提高训练稳定性。

模型输出目录默认位于 Hugging Face 缓存目录下的 \path|models/qwen2_5_sft_cot_lora|。脚本支持从 checkpoint 自动恢复训练，并每隔固定步数保存 checkpoint。若未显式提供验证集，脚本会按默认比例从训练集中划分少量样本作为验证集，用于监控 eval loss 和选择最佳模型。

\section{基础模型、SFT模型与RL模型关系}

本文的模型迭代关系可以分为三个层次。第一层是基础模型，即未经任务微调的 Qwen2.5-7B-Instruct。基础模型具备通用语言理解和金融文本分析能力，但在股票方向预测任务中，其输出格式、推理重点和最终标签稳定性并未专门适配。第二层是 SFT 模型，即在基础模型上加载 LoRA adapter 后得到的任务适配模型。SFT 模型通过大量“输入—输出”样本学习如何理解归一化行情特征、遵循预测格式，并在最后输出可解析的涨跌方向。

第三层是预留的 RL 模型。RL 模型不是从零开始训练，而是在 SFT 模型已经具备基本任务能力之后，进一步利用奖励或偏好信号进行优化。也就是说，基础模型提供通用语言能力，SFT 模型完成任务格式和领域样本适配，RL 模型则面向最终评价目标进行进一步对齐。由于本文当前实验主线以监督微调和离线评估为主，强化学习部分暂作为后续扩展路径保留。

在评估实现上，本文使用 \path|train/scripts/sft/eval_sft_vllm_compare.py| 对基础模型和 SFT LoRA 模型进行对比。评测脚本使用 vLLM 加载基础模型，并通过 LoRARequest 动态加载微调 adapter；随后对同一批 eval prompts 分别生成基础模型输出和微调模型输出，再从模型回复中提取最终的“涨”或“跌”标签，与真实标签进行比较。对于多源信息样本，\path|test/eval_quotes_summary_5d_vllm.py| 则按照 1、3、7 日三个位置分别计分，计算平均方向预测得分。

\section{本章小结}

本章介绍了本文模型训练与优化方法。首先，本文将股票短期预测任务定义为涨跌方向分类任务，而非精确涨跌幅回归任务，以提高监督标签稳定性和实验可评估性。其次，本文基于 \path|exports/quotes.csv| 构建 7 日 K 线监督数据，并进一步通过批量大模型生成方式构建 CoT 风格训练样本；同时，本文也设计了融合新闻和研报摘要的多源样本，用于分析外部信息对 1、3、7 日预测的影响。最后，本文采用 LoRA 对 Qwen2.5-7B-Instruct 进行监督微调，并通过 vLLM 对基础模型和微调模型进行对比评估。强化学习部分当前暂作为后续扩展方向保留。

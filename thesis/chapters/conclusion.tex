\chapter{总结与展望}

\section{全文工作总结}

本文围绕股票短期走势预测中“结构化行情信息不足以反映实时市场预期变化”的问题，设计并实现了一个基于大语言模型的多源信息融合预测框架 MarketMind。该框架面向真实投研场景，在预测时综合利用历史行情、投资者评论、新闻报道、机构研报以及外部检索信息，将多源异构文本经过筛选、汇总和统一组织后输入大语言模型，由模型完成市场信息归纳、利好利空因素识别、跨源证据整合以及未来短期走势方向判断。

在理论与技术基础方面，本文首先梳理了股票短期方向预测、金融文本分析、检索增强生成、大语言模型、Prompt 设计、监督微调与强化学习对齐等相关内容。股票短期走势预测不是单纯的时间序列拟合任务，而是同时受到事件冲击、投资者情绪、市场关注度和信息扩散速度影响的复合型任务。因此，本文将其建模为“金融语义理解、多源信息融合与方向决策”相结合的问题，并将大语言模型作为统一的信息理解与预测核心。

在系统框架方面，本文结合仓库中的实际脚本设计了从数据采集到模型预测的完整流程。系统通过评论、新闻、研报等不同采集模块获取近期市场信息，对内容进行时间窗口控制、去噪、截断和摘要处理，并围绕股票代码、日期、历史 K 线和文本摘要构造统一 Prompt。模型最终输出未来 1 日、3 日和 7 日的涨跌方向判断及相应依据。该流程强调预测时的信息可见性，尽量避免离线实验中常见的未来信息泄漏问题。

在模型训练方面，本文以 Qwen2.5-7B-Instruct 为基础模型，构建了面向股票方向预测的监督微调数据。训练样本以历史 K 线窗口和真实未来涨跌标签为基础，并进一步通过批量生成方式构造包含分析过程的 CoT 风格样本。模型训练采用 LoRA 监督微调方式，在控制训练成本的同时提升模型对股票方向预测任务格式和金融语义的适配能力。同时，本文也对后续引入 GRPO 等强化学习方法进行策略优化的可行路径进行了设计说明。

在实验验证方面，本文分别进行了微调前后对比实验和有无外部信息对比实验。实验结果表明，在仅使用历史 K 线的 7 日单周期任务中，LoRA 监督微调模型相较基础模型取得了一定提升；在多周期任务中，引入新闻和研报摘要后，模型整体预测效果优于仅依赖 K 线的设置，尤其在未来 1 日和 3 日方向预测上体现出更明显的增益。这说明外部文本信息能够为短期市场方向判断提供行情数据之外的补充信号，也验证了本文多源信息融合框架的基本有效性。与此同时，本文还在在线系统中实现了基于 SerpAPI 的 Web 检索补充模块，用于在实时查询时进一步汇聚公开网页信息。

\section{主要研究结论}

通过本文的系统设计与实验分析，可以得到以下结论。

第一，面向股票短期走势预测，仅依赖历史价格和成交量等结构化行情信息存在明显局限。短期价格变化往往受到新闻事件、市场情绪和投资者预期变化影响，而这些因素更多体现在非结构化文本中。将评论、新闻和研报等外部信息纳入预测流程，能够使模型获得更丰富的市场上下文。

第二，大语言模型适合作为多源金融信息的统一理解与归纳工具。相比传统手工情绪指标或单一文本分类方法，大语言模型能够在同一框架下完成事件提取、观点总结、利好利空识别和跨源一致性分析。本文实验结果也表明，在合理的 Prompt 组织和输出约束下，大语言模型能够将多源文本转化为可用于方向判断的结构化依据。

第三，监督微调能够提升模型在金融预测任务上的适配能力。基础指令模型虽然具备一定语言理解能力，但在具体股票预测任务中容易受到输出格式、任务边界和领域语义的限制。通过 LoRA 监督微调，模型能够更稳定地理解输入格式、遵循预测要求，并在离线测试中取得优于基础模型的结果。

第四，多源外部信息对短期预测具有实际增益。实验中，加入新闻和研报摘要后，模型在多周期任务中的有效样本平均准确率高于仅 K 线输入设置，说明外部文本能够补充历史行情中难以直接体现的事件和预期信息。其中，未来 1 日和 3 日预测提升更为明显，符合短期市场更容易受即时信息影响的特点。系统实现层面进一步接入了 SerpAPI，以多 query 检索、统一去重合并的方式补充最新公开网页结果，为后续扩展到更实时的在线预测场景提供了信息入口。

第五，面向真实应用的金融大模型系统不能只关注模型本身，还需要关注数据管线、信息时效、输入组织、输出解析和评估方式。本文构建的 MarketMind 框架从工程角度打通了信息采集、摘要汇总、Prompt 构造、模型调用和结果评估流程，为后续构建更完整的智能投研系统提供了基础。

\section{本文不足}

尽管本文完成了多源信息融合股票预测框架的设计、实现与初步验证，但仍存在以下不足。

首先，当前实验规模和评估时间窗口仍然有限。股票市场具有明显的阶段性特征，不同年份、不同市场风格和不同板块中的信息有效性可能存在差异。本文实验主要基于已有数据和特定时间区间进行离线验证，尚不足以全面证明模型在长期、多市场环境下的泛化能力。

其次，外部信息实验仍需要更严格的单因素消融。当前多源信息实验已经显示外部文本输入能够带来性能提升，但部分实验设置中同时包含模型微调和输入信息变化，因此尚不能完全分离“模型参数优化”和“外部信息接入”各自的贡献。后续需要在完全相同模型、相同解码参数和相同样本集合下，分别测试仅 K 线、K 线加新闻、K 线加研报以及完整多源输入的效果。

再次，文本信息质量控制仍有提升空间。评论、新闻和研报来源具有不同噪声水平，尤其是评论信息中可能包含重复表达、情绪宣泄或误导性内容。本文主要通过时间窗口、截断和摘要方式进行初步处理，尚未充分引入更细粒度的可信度评估、来源权重建模和虚假信息过滤机制。

此外，模型输出的稳定性仍需进一步增强。多源输入会增加上下文长度和任务复杂度，实验中也出现了部分输出无法被自动解析的情况。这说明仅依靠自然语言 Prompt 约束仍不够稳健，后续可以采用更严格的 JSON schema、结构化解码、函数调用式输出或分类头约束来降低格式错误。

最后，本文目前重点验证的是方向预测能力，还没有完整进入交易决策层面。真实金融实践不仅需要判断涨跌方向，还需要考虑收益风险比、仓位控制、交易成本、流动性、止损机制和回撤约束。因此，当前结果更适合作为智能投研辅助信号，而不能直接等同于可实盘执行的交易策略。

\section{后续研究展望}

未来工作可以从以下几个方向进一步展开。

第一，扩展数据规模与评估范围。后续可以引入更长时间跨度、更多股票池和更多市场阶段的数据，并按照牛市、熊市、震荡市、行业轮动和事件驱动行情进行分层评估，从而更准确地分析模型在不同市场环境下的适用边界。

第二，完善多源信息消融实验。后续应在同一模型条件下分别评估 comments、news、reports 和 Web 检索结果的独立贡献，并进一步分析不同信息源在不同预测周期中的作用差异。例如，评论信息可能更适合捕捉 1 日内情绪变化，研报信息可能更适合解释较长周期的基本面逻辑，而 Web 检索结果则可能在突发事件中发挥更强补充作用。

第三，提升检索增强与证据组织能力。当前系统已经完成 SerpAPI 实时接入以及多 query 结果的 URL 去重合并，但该部分仍主要服务于在线信息展示，尚未形成可稳定回放的离线评测集。后续可以在现有实现基础上继续引入可信度排序、时间衰减、证据引用和历史快照存档机制，使模型不仅能够生成预测结论，还能够给出可追溯的证据来源，降低金融大模型在事实性和可解释性方面的风险。

第四，改进模型输出约束与评价体系。未来可以将自然语言输出进一步结构化，例如要求模型以固定 JSON 字段输出方向、置信度、核心证据和风险因素；同时增加 F1、收益模拟、最大回撤、信息系数等指标，从分类准确率和投资实践效果两个角度共同评价模型。

第五，引入强化学习进行策略级优化。在监督微调基础上，后续可以探索 GRPO 等强化学习方法，将预测正确性、理由一致性、风险提示质量和策略收益表现设计为奖励信号，使模型从“按格式完成方向判断”进一步过渡到“在约束条件下优化预测与决策质量”。这一方向有助于把本文系统从信息聚合与方向预测扩展为更完整的智能投研与辅助决策平台。

总体来看，本文完成了 MarketMind 的初步框架构建与实验验证，证明了大语言模型结合多源金融文本信息在股票短期方向预测中的应用潜力。随着实时数据接入、检索增强、领域微调和强化学习优化的进一步完善，该类系统有望在金融信息分析、市场共识发现和智能投研辅助中发挥更大的价值。

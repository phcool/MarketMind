\chapter{相关技术与理论基础}

\section{股票短期走势预测任务概述}

股票短期走势预测是量化投资与金融智能分析中的重要任务，其核心目标是在给定历史信息与当前市场状态的条件下，对未来较短时间窗口内的价格变动方向或收益变化进行判断。与中长期价值分析相比，短期走势更容易受到事件冲击、投资者情绪、市场关注度和信息扩散速度等因素影响，因此仅依赖历史价格与成交量等结构化数据，往往难以充分刻画短期波动背后的驱动机制。已有研究表明，新闻文本、社交媒体情绪以及其他非结构化信息对股票价格短期变化具有一定预测价值，大语言模型也逐渐展现出从金融文本中提取有效市场信号的能力\cite{bollen2011twitter,zhang2025gnn,zhou2025expertopinion,lopezlira2023chatgptstock,elahi2024financialnewsllm}。

从任务形式上看，股票短期走势预测既可以建模为收益率回归问题，也可以建模为涨跌方向分类问题。考虑到真实市场中的短期收益分布噪声较大、极易受到偶然因素干扰，而方向判断更符合实际交易决策中的“买入—观望—卖出”逻辑，本文将任务定义为面向未来 1 日、3 日和 7 日的涨跌方向预测任务。该任务不仅需要模型理解历史价格所反映的市场状态，还需要综合评论、新闻、研报以及实时 Web 检索结果等外部信息，判断当前时点下市场叙事、风险事件和投资预期可能对未来走势产生的影响。

与传统时间序列模型不同，基于大语言模型的股票短期预测任务具有更强的语义建模属性。模型不再只是拟合价格序列的统计规律，而是需要在多源异构文本中抽取事件、观点、情绪、因果线索和潜在共识，再将这些信息映射为未来价格方向判断。因此，这一任务可以理解为“金融语义理解 + 多源信息融合 + 方向决策”的复合型任务，其理论基础涉及金融文本分析、检索增强生成、监督微调与强化学习对齐等多个方向\cite{lewis2020rag,ye2024r2ag,lee2024finllms,li2023llmfinance,nie2024llmfinancialapplications,yang2023fingpt,wu2023bloomberggpt}。

\section{多源金融信息的特点与作用}

金融市场中的信息来源具有显著的异构性。不同来源在发布时间、文本风格、噪声水平、专业性和影响路径上存在明显差异。对于短期走势预测而言，评论信息更接近即时情绪，新闻信息更体现事件冲击，研报信息更强调逻辑分析，而 Web 检索结果则能够起到外部补充与交叉验证作用。如何理解这些信息源的差异，并建立统一的信息组织方式，是构建多源预测框架的前提\cite{lee2024finllms,li2023llmfinance,nie2024llmfinancialapplications}。

\subsection{评论信息的时效性与情绪性}

评论信息主要来自股吧、论坛、社交媒体和投资社区，其最大特点是更新快、情绪强、噪声大。在市场出现热点题材、突发消息或板块异动时，评论通常比正式新闻更早反映投资者关注点和态度变化。这类信息能够快速揭示市场情绪、风险偏好和群体预期变化，因此对于短期走势预测尤其重要。Bollen 等研究发现，社交媒体中的群体情绪与市场变化之间存在统计关联，为评论型文本进入金融预测提供了早期证据\cite{bollen2011twitter}。

但评论信息并不等同于有效信号。其内容中往往夹杂大量重复表达、情绪宣泄、非理性观点甚至误导性信息。因此，如果直接将评论原文输入模型，容易放大噪声干扰。后续研究进一步指出，社交媒体和论坛的真正价值不在于原始帖子数量本身，而在于能否识别出持续提供有效判断的用户、代表性观点及其扩散关系。基于图结构的情绪分析方法和专家意见挖掘方法均说明，对评论信息进行筛选、归纳和结构化处理，比简单做情绪正负分类更能提升预测价值\cite{zhang2025gnn,zhou2025expertopinion}。

对本文而言，评论信息的作用主要体现在两个方面：一是补充传统价格序列无法直接体现的市场情绪变化；二是帮助模型捕捉题材热度、散户预期与短期交易叙事。由于评论噪声较高，本文不直接依赖手工设计情绪公式，而是通过大语言模型对近期评论进行总结、归纳和冲突识别，使其成为更适合预测任务的结构化输入。

\subsection{新闻信息的事件驱动特征}

与评论信息相比，新闻信息具有更强的事件驱动属性。公告、政策、行业动态、公司经营变化、国际局势和突发风险事件，往往首先通过新闻媒体被市场广泛感知，并进一步影响投资者预期。短期股价波动中，很多显著变动都与特定事件直接相关，因此新闻文本是短期预测中不可忽视的重要来源。已有研究表明，大语言模型能够从新闻标题和报道中识别对未来收益具有解释力的关键信息，这说明新闻并不仅仅提供事实，还能承载影响价格方向的市场叙事与因果线索\cite{lopezlira2023chatgptstock,elahi2024financialnewsllm}。

新闻信息的优点在于可信度相对较高、文本结构相对规范、事件边界较清晰，但其局限性在于：第一，新闻通常滞后于最早的市场讨论；第二，不同媒体的报道角度可能不同；第三，单条新闻的影响强度往往依赖历史背景与其他信息源共同解释。因此，在实际任务中，新闻信息不宜被孤立使用，而应与评论、研报和检索结果联合建模。本文将新闻视为“事件触发型证据”，用于帮助模型识别短期走势中的关键外部驱动因素。

\subsection{研报信息的专业分析特征}

研报信息主要来自券商、研究机构或专业分析团队，其突出特点是专业术语多、逻辑链条完整、分析视角相对系统。与评论和新闻相比，研报通常更关注行业逻辑、公司基本面、估值判断、政策影响和未来催化剂，因此在解释“为什么涨”“为什么跌”方面具有更强的分析深度。金融领域相关研究和综述普遍指出，金融文本具有鲜明的领域术语和上下文依赖，普通通用模型往往难以充分理解其中的专业含义，这也是金融专用模型和金融领域微调持续受到重视的重要原因\cite{araci2019finbert,lee2024finllms,li2023llmfinance,nie2024llmfinancialapplications,wu2023bloomberggpt}。

研报信息的局限在于覆盖面和时效性并不总是最强，而且不同机构的观点可能带有自身研究框架和立场偏好。因此，研报更适合作为“高质量分析证据”来增强模型的逻辑判断能力，而不是作为唯一信息来源。对于本文任务而言，研报的主要价值在于为模型提供更专业的行业解释和未来预期分析，从而在评论和新闻之外增加一层较高质量的推理依据。

\subsection{Web 检索结果的补充作用}

在真实预测场景中，单纯依赖预先收集的静态数据往往不足以反映市场最新变化。尤其是在个股或板块出现临时热点、政策变化或舆情波动时，模型若无法接入实时外部信息，就很容易受到参数化知识陈旧性的限制。RAG 研究表明，通过在生成过程中接入外部检索文档，可以显著提升模型在知识密集任务中的事实性与更新能力\cite{lewis2020rag}。R\textsuperscript{2}AG 则进一步指出，仅仅“检索到文档”还不够，还需要通过更合理的提示设计和检索信息组织方式，帮助模型真正理解并利用检索结果\cite{ye2024r2ag}。

因此，Web 检索结果在本文框架中的作用主要体现在三个方面。第一，它能够补充评论、新闻和研报未覆盖到的最新事实信息；第二，它能够为模型提供交叉验证材料，降低单一来源偏差带来的误判风险；第三，它能够帮助模型扩展上下文视野，使其在面对新事件、新概念或新市场热点时保持更强的适应能力。也正因为如此，本文在真实应用设定中将 Web API 检索作为预测流程的重要组成部分，而不仅仅把它看作可有可无的附加模块。

\section{大语言模型基本原理}

大语言模型的核心基础是 Transformer 架构。Vaswani 等提出的 Transformer 通过自注意力机制建模序列中不同位置之间的依赖关系，摆脱了传统循环网络对顺序计算的依赖，为大规模并行训练和长距离语义建模提供了基础\cite{vaswani2017attention}。在此基础上，现代大语言模型通常采用大规模无监督预训练方式，通过海量文本学习词语、句子、段落以及更高层次语义之间的统计规律，从而获得通用语言理解与生成能力。

从训练流程上看，大语言模型通常经历“预训练—指令对齐—任务适配”三个阶段。预训练阶段主要让模型获得广泛的语言知识与世界知识；指令对齐阶段则通过监督微调或强化学习等方式，使模型更好理解用户指令并输出符合任务要求的回答；任务适配阶段则面向具体领域或具体任务进一步优化模型表现。Ouyang 等在 InstructGPT 中系统展示了“监督微调 + 偏好对齐”的训练流程，说明语言模型不仅要会生成文本，还要学会按照人类期望方式完成任务\cite{ouyang2022instructgpt}。

在金融场景中，大语言模型的价值不仅来自通用语言能力，更来自其对复杂金融语义、跨文本证据整合和高层次推理的支持。BloombergGPT、FinGPT 以及多篇金融 LLM 综述都表明，金融任务具有专业术语密集、时效性强、事实约束高等特点，因此模型在该领域的应用通常需要额外的数据构建、检索接入和任务级微调\cite{lee2024finllms,li2023llmfinance,nie2024llmfinancialapplications,yang2023fingpt,wu2023bloomberggpt}。这意味着，在股票短期走势预测任务中，大语言模型既是语义理解器，也是多源信息融合器，更是最终的方向判断器。

\section{基于 Prompt 的金融文本理解方法}

Prompt 方法是将复杂任务转换为模型可直接理解的自然语言指令的一种关键技术路径。对于金融文本而言，Prompt 不仅决定模型“看什么”，还决定模型“如何看”。同样一批评论、新闻和研报，如果没有合理的任务说明与输出约束，模型可能只会生成泛泛而谈的总结；而通过明确要求其识别利好利空因素、提炼核心事件、分析不同来源之间的一致与冲突，并最终输出方向判断，模型就更容易形成面向预测任务的结构化理解\cite{lee2024finllms,li2023llmfinance,nie2024llmfinancialapplications}。

在金融应用中，Prompt 方法尤其适合处理弱结构化、多来源、强语境依赖的文本输入。FinGPT 等工作表明，金融大模型可以借助任务指令和轻量微调完成情感分析、信息抽取、问答和预测等多类任务\cite{yang2023fingpt}。同时，ChatGPT 对新闻文本的收益预测能力以及多模态检索增强方法的效果也说明，只要给出合理的问题定义和输入组织方式，大语言模型就能够从复杂金融语义中抽取与市场变化相关的有效信息\cite{lopezlira2023chatgptstock,elahi2024financialnewsllm}。

结合本文任务，基于 Prompt 的金融文本理解主要包括三个层次。首先，对评论、新闻和研报分别进行局部总结，使模型在每个来源内部提炼核心观点和主要证据；其次，将三类总结与 Web 检索结果共同输入综合 Prompt，引导模型比较不同来源之间的一致性和差异性；最后，要求模型输出未来 1 日、3 日和 7 日的涨跌判断及其依据。这样的设计有两个优点：一方面可以降低长文本和噪声对模型的干扰；另一方面可以提升输出结果的可解释性和可审查性。与 RAG 和 R\textsuperscript{2}AG 的思想一致，这里 Prompt 的作用不仅是“提问”，更是对外部信息进行重组和引导推理\cite{lewis2020rag,ye2024r2ag}。

\section{监督微调方法}

监督微调（Supervised Fine-Tuning, SFT）是当前大语言模型任务适配中最常用的方法之一，其基本思想是在已有预训练模型基础上，利用高质量“输入—输出”样本对模型进行进一步训练，使其逐步学会按照指定格式、指定任务目标和指定领域知识完成输出。相较于从头训练，监督微调成本更低、落地更快，也更适合金融领域这样专业语义较强但可用标注数据相对有限的应用场景\cite{yang2023fingpt,ouyang2022instructgpt}。

在金融任务中，监督微调的主要作用有三点。第一，它能够让模型更熟悉金融文本中的专业表达与分析逻辑，从而提升对评论、新闻和研报的理解能力；第二，它能够让模型学会更加稳定的输出格式，例如固定输出未来涨跌方向、核心依据和风险提示；第三，它能够在一定程度上减少通用模型在金融任务上的幻觉和不稳定表达。FinBERT 和 FinGPT 等研究都说明，领域适配和任务微调对于金融文本理解性能提升具有实际意义\cite{araci2019finbert,yang2023fingpt}。

本文的监督微调目标并不是让模型记忆某些静态规则，而是让其学会在给定多源信息摘要后，输出更符合股票短期预测任务需求的方向判断。也就是说，SFT 在本文中承担的是“任务格式对齐 + 金融语义适配”的作用。通过微调，模型能够更稳定地将外部信息映射为 1 日、3 日和 7 日的涨跌判断，这也为后续进一步引入强化学习优化提供了基础。

\section{强化学习对齐方法（GRPO）}

在完成监督微调后，模型虽然已经具备较好的任务执行能力，但其输出结果未必能够在更高层面上与最终目标保持一致。对于股票短期走势预测而言，仅仅学会“按格式回答”还不够，模型还需要在多种可能输出中逐步偏向那些更有助于提升预测质量、逻辑一致性和稳健性的回答。因此，在 SFT 之后继续采用强化学习进行对齐，是一种自然的扩展路径。InstructGPT 的研究表明，在监督微调之后引入偏好信号或奖励信号，可以进一步提升模型输出与目标要求之间的一致性\cite{ouyang2022instructgpt}。

传统强化学习对齐中，PPO 是最常见的方法之一。PPO 通过限制策略更新幅度来提升训练稳定性，在强化学习和 RLHF 中都得到了广泛应用\cite{schulman2017ppo}。但在大语言模型场景下，PPO 往往需要额外的价值模型（critic）来估计基线，这会增加显存和训练复杂度。尤其当主模型规模较大时，critic 的引入会显著提高训练成本，使强化学习阶段变得更难部署和调试。

GRPO（Group Relative Policy Optimization）是 DeepSeekMath 中提出的一种 PPO 变体，其核心思想是在同一问题上生成一组候选输出，并基于组内相对表现来构造学习信号，而不是显式训练一个独立的 critic 模型。DeepSeekMath 指出，GRPO 通过利用组内得分估计基线，能够在保持强化学习优化能力的同时降低 PPO 的内存消耗，更适合大模型训练场景\cite{shao2024deepseekmath}。

从本文任务出发，GRPO 的适用性主要体现在以下几个方面。第一，股票方向预测本身允许同一输入下生成多个候选判断，因此天然适合组内比较式优化；第二，相较于传统 PPO，GRPO 不依赖额外价值模型，更适合在有限算力条件下进行大模型强化学习；第三，本文关注的不只是答案表面形式，而是预测是否更准确、理由是否更一致、是否更能利用多源信息，这些目标都可以通过设计奖励规则或偏好信号逐步注入训练过程。因此，在本文的后续研究中，GRPO 可被视为从“监督学习的静态拟合”过渡到“面向预测效果的动态优化”的关键方法。

需要说明的是，本文引入 GRPO 的目的并不是把股票预测问题简单等同于标准强化学习任务，而是在监督微调已具备基本能力的前提下，进一步利用组内比较与奖励信号优化模型输出质量。换言之，SFT 负责让模型“学会做题”，而 GRPO 更侧重让模型“做得更好、更稳定、更符合最终评价目标”。

\section{本章小结}

本章围绕本文方法所涉及的关键技术与理论基础进行了梳理。首先，股票短期走势预测本质上是一个融合金融语义理解与方向决策的复合型任务，单纯依赖结构化行情数据难以完整捕捉短期波动的成因。其次，评论、新闻、研报与 Web 检索结果在时效性、专业性、事件性和补充性方面各具优势，多源融合是提升预测效果的重要前提。再次，大语言模型为金融文本理解、多源信息整合和方向判断提供了统一建模框架，而 Prompt 设计则决定了模型如何将复杂文本转化为面向预测任务的有效表示。最后，监督微调能够提升模型对金融任务的适配能力，GRPO 则为进一步优化模型输出质量和预测性能提供了强化学习路径。以上内容共同构成了本文后续系统设计、实验构建与方法实现的理论基础。
